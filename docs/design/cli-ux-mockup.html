02Help System and Commands

Built-in help with contextual command suggestions and clear documentation.

healthsim - zsh - 100x30
You: /help

📦 Generation

  • "Generate 1000 members in California"
  • "Add 12 months of claims history"
  • "Generate pharmacy claims for diabetics"
  • "Create a patient with CHF and diabetes"

📊 Analytics

  • "Profile this population" - demographics, conditions, utilization
  • "Stratify risk, 4 tiers" - assign risk levels
  • "Define cohort for [criteria]" - create named subgroups
  • "Compare [cohort A] vs [cohort B]" - cohort comparison
  • "What are the cost drivers?" - cost analysis

🗃 Database

  • "Show tables" - list generated data tables
  • "Describe [table]" - show table schema
  • "/sql SELECT ..." - execute custom query

📥 Export

  • "Export to FHIR Bundle" - clinical data
  • "Export to X12 837P" - professional claims
  • "Export to CSV" - tabular data

⚙ System

  • /help - show this message
  • /clear - clear screen
  • /cost - show session cost
  • /exit or quit - exit application

05UX Design Principles

Core principles embedded in every interaction.

🧭 Guided Discovery

Every response suggests logical next steps. Users are never stuck wondering "what now?"

💡 Proactive Insights

Surface interesting patterns automatically - care gaps, cost outliers, high-risk cohorts.

📊 Progressive Depth

Headlines first, offer to drill down. Show the forest, then the trees on request.

🔗 Connect the Dots

Reference previous findings. "This cohort accounts for 21% of costs you saw earlier."

🔧 Tool Transparency

Show tool execution (-> generate_members) so users understand what's happening.

💰 Cost Awareness

Display API cost per interaction. Users can manage their budget consciously.
